# 用户使用与可访问性建议

本指南整理了使用与体验层面的建议，帮助在不同设备上获得更好的交互与隐私保护。

## 可用性与可访问性

- 响应式布局：在桌面与移动设备均保持可用与清晰的布局。
- 键盘可访问：表单与对话框支持 `Tab/Shift+Tab` 切换焦点与 `Enter` 提交。
- 颜色对比：建议遵守 WCAG AA 等级，避免低对比度文本与关键按钮不可见。
- 替代文本：头像、图表等非文本内容提供 `alt` 或说明文字，便于读屏软件识别。
- 动效控制：避免频繁闪烁，为用户提供减少动效的选项以降低视觉负担。

## 隐私与安全

- 最小化收集：仅保存健康数据所需字段，不采集与业务无关的信息。
- 令牌存储：避免将敏感令牌持久化到 `localStorage`；优先使用受控 Cookie 或内存。
- 传输安全：生产环境采用 `HTTPS` 与安全的反向代理配置。
- 上传限制：头像与文件上传限制类型与大小，服务端校验通过后再入库。
- 数据导出：CSV 导出仅包含当前用户可访问数据，不暴露他人信息。

## 常见问题（FAQ）

- 登录失效提示：出现“未登录或登录已过期，请重新登录”为鉴权失败的正常提示，请重新登录。
- CSV 乱码：使用支持 `UTF-8` 的编辑器或在 Excel 导入时选择 `UTF-8` 编码。
- 图表不显示：可能当前时间范围无数据，调整筛选条件或检查记录是否成功保存。
- 头像不更新：浏览器可能缓存了头像，可尝试强制刷新或稍候重试（系统有头像存在性缓存）。

## 性能与体验建议

- 分页与懒加载：列表页建议分页加载并按需懒加载长列表元素。
- 图片优化：优先使用压缩图片与 `webp` 等现代格式。
- 合理的页大小：每页 `10-50` 条更易于浏览与渲染，避免超大 `pageSize`。
- 前后端校验：前端进行基础校验，后端作为兜底校验，减少无效请求。
